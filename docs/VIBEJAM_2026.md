## Rules (as communicated)

- Anyone can enter with their game.
- **At least ~90% of code** should be written by AI.
- Game must be **started today or after** — do not submit old games.
- **Web:** playable in the browser **without login or signup**, **free to play**; **prefer** own domain or subdomain.
- **Multiplayer** is preferred but not required.
- **Engine:** any allowed; **Three.js** is commonly recommended.
- **Performance / UX:** **No loading screens and no heavy downloads** — players should be **in the game almost instantly** (optional: e.g. ask for username).
- **Entry:** include the **HTML snippet** from the Google form reply in the submission.
- **One entry per person** — focus on one strong game.

## This project

- Stack: **Vite + TypeScript + Three.js**, `npm`.
- Jam constraints inform asset size, no splash gating, and keeping the first frame fast.
